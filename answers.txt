1) Збережений (commited) - дані безпечно збережено в локальній базі даних; 
Змінений (modified) - у файл внесено редагування, які ще не збережено в базі даних;
Індексований (staged) -виникає тоді, коли ви позначаєте змінений файл у поточній версії, щоб ці зміни ввійшли до наступного знімку коміту.
2)команда add використовується для додавання будь-яких нових або змінених файлів в індекс для наступного комміта.
3)Команда status показує поточний стан файлів відносно моєї робочої області і репозиторія Git.
4)Команда commit використовується для збереження змін у моєму локальному Git репозиторії.
5)Команда init використовується для підключення системи контролю версій до мого проекту (або папки).
6) В коментарях до комітів потрібно писати опис зробленої роботи
7) Команда git checkout використовується для створення,переходу між гілками відміни змін
8)Команда reset використовується для відміни змін
9) Команда log використовується для переліку комітів зроблених в репозиторії в оберненому щодо хронологічного порядку. 
Останні коміти знаходяться зверху
10)Команда status показує поточний стан файлів відносно моєї робочої області і репозиторія Git.
11) Щоб відновити кілька файлів з попереднього коміта використовуємо команду git reflog
12) Щоб перейти до попереднього коміта по імені використовуємо команду git checkout назва гілки
13) Нову команду можна створити за допомогою команди git branch а щоб і перейти в неї git checkout -b 
14) Щоб перейти на нову гілку використовуємо команду git checkout
15) Щоб перейти на основну гілку git checkout master
16) Щоб виконати злиття гілки використовуємо команду git merge
